## 计算属性

>**计算属性**：根据现有的数据（响应式数据、组件属性），计算得来的新数据。
>
>- 计算属性本质是一个拥有`getter`和`setter`方法的访问器属性。
>- 首次访问计算属性，会执行计算属性的`getter`方法。在getter方法内：
>  - 分析并记录依赖的数据（响应式数据、组件属性）。
>  - 将计算结果缓存，往后访问直接从缓存中获取。
>- 当计算属性依赖的数据发生改变后，计算属性会再次调用其`getter`方法，更新缓存结果。保证往后计算属性能从缓存中获取最新的计算结果。
>- 计算属性在依赖数据不改变的情况下，仅会执行一次`getter`方法，其后从缓存中获取计算结果。

**注意**：计算属性是一个数据。具备可读可写的能力。

**不推荐**：在计算数据的过程中使用**异步**、**当前时间**、**随机数**等副作用操作。



1. **完整的计算属性定义**：

   ```js
   const Comp = {
       computed: {
           propName: {
               get(){
                   // getter方法
               },
               set(val){
                   // setter方法
               }
           }
       }
   };
   ```

   

2. **只读的计算属性定义（常用）**：

```js
const Comp = {
    computed: {
        propName(){
            // getter方法
        }
    }
};
```





## 方法

>在`methods`中定义的方法，是普通方法。
>
>- 计算结果不会缓存。
>- 每调用一次执行一次。

**注意**：方法是一个操作。
